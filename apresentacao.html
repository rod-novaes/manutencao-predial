O Desafio

Complexidade

Rastrear centenas de serviços com múltiplas periodicidades (semanal, mensal, anual) de forma manual.

Planejamento

Elaborar manualmente planos de trabalho, consumindo tempo valioso da fiscalização técnica.

Mitigação de Riscos

Trabalhar com processos manuais aumenta o risco de desvios e esquecimentos, que pode impactar na conformidade com o PMOC.

A Solução Proposta

Digitalizar o PMOC

Converter as regras, checklists e periodicidades do plano em um sistema inteligente, que serve como uma extensão digital do plano oficial.

Centralizar a Base

Criar uma base de dados única para todas as grandes áreas, sistemas, componentes e históricos de serviços, consolidando a informação.

Automatizar o Plano

Delegar ao sistema a tarefa de calcular, agendar e organizar as manutenções futuras, gerando relaórios de trabalho precisos e automáticos.

Foco Estratégico: Esta é uma ferramenta de planejamento e controle para a COSEN. Ela não interfere nem substitui sistemas oficiais de OS (IBM Máximo), mas pode os alimentar dentro do interesse da coordenadoria.

Funcionalidades

Dashboard Visual

Painel inicial que agrupa e prioriza os serviços vencidos, do mês atual e do próximo, com um calendário interativo.

Estrutura Fiel ao PMOC

Organização em Grandes Áreas, Sistemas e Componentes, refletindo a lógica do plano oficial.

Agendamento por Serviço

Cada serviço possui sua própria periodicidade, garantindo conformidade com as especificações técnicas do PMOC.

Flexibilidade de Cadastro

Permite adicionar serviços específicos para um componente individual, além dos serviços herdados do sistema padrão.

Geração Automática

O sistema calcula os próximos vencimentos com base no histórico, mantendo o cronograma automaticamente atualizado.

Relatórios de Trabalho

É possível gerar um plano de trabalho formatado para qualquer período, pronto para ser entregue às equipes terceirizadas ou para servir de base de comparação com o Relatório Técnico de Manutenção recebido.

Fluxo de Trabalho Proposto

1. Planejar

A COSEN alimenta o sistema com os dados do PMOC e o inventário de serviços.

2. Gerar

O gestor gera o relatório de serviços pendentes para o período desejado.

3. Executar

O relatório é entregue à contratada como escopo do serviço.

4. Registrar

O fiscal registra a conclusão, e o sistema agenda o próximo ciclo automaticamente.

Vantagens da Adoção

  • Mitigação de Riscos: Reduz significativamente o risco de esquecimento de serviços críticos, garantindo maior conformidade com o PMOC.
  • Otimização do Tempo: Automatiza o processo de planejamento, liberando a equipe técnica para focar na fiscalização, análise e melhoria contínua.
  • Padronização: Garante que o escopo de trabalho comunicado às contratadas seja completo, preciso e uniforme.
  • Histórico Confiável: Constrói uma base de dados estruturada para auditorias e tomadas de decisão futuras.

Limitações e Oportunidades Futuras

  • Arquitetura Standalone: O aplicativo foi projetado para operar sem depender da infraestrutura de servidores do TSE. Os dados do protótipo são salvos no navegador.
  • Aplicação Monousuário: O protótipo não é multiusuário e não possui sistema de autenticação, sendo destinado ao uso em uma estação de trabalho por vez.
  • Fluxo Manual: A integração com sistemas externos (ex: IBM Máximo) pode ser feita via registro manual pelo fiscal, se isso for do interesse da Cosen.
  • Oportunidade de Evolução: Caso o desenvolvimento seja aprovado, podem ser implementados recursos como gestão de dados local (Salvar/Abrir/Backup) e dashboards de KPIs.
