Visão Geral do Sistema

Digitalização do PMOC

O sistema funciona como uma versão digital e interativa do Plano de Manutenção, Operação e Controle. Ele converte as listas de serviços e suas periodicidades em uma base de dados estruturada.

Automação do Planejamento

Sua função primária é automatizar o cálculo e a organização das manutenções futuras. Com base no histórico de execuções, o sistema projeta o cronograma, eliminando a necessidade de elaboração manual.

Centralização do Histórico

Cada serviço executado é registrado, construindo um histórico centralizado por componente, sistema, subsistema ou grande área. Isso serve como uma fonte de dados para análises e auditorias.

A Estrutura de Dados: O Coração do Sistema

Hierarquia Lógica

Para refletir a organização de um plano de manutenção real, o sistema utiliza uma estrutura hierárquica. Entender essa lógica é fundamental para operá-lo eficientemente.

Grande Área (Ex: Elétrica, Climatização)
Sistema (Ex: Grupo Geradores, Sistema VRF)
Subsistema (Opcional) (Ex: Condensadoras, Evaporadoras)
Componente (Ex: Gerador Stemac 01, Condensadora TAG 1A)

Herança de Serviços

O poder desta estrutura está na herança de checklists:

  • Você define serviços de manutenção no nível do Sistema.
  • Todos os Componentes associados a esse Sistema automaticamente herdam essa lista de serviços.
  • É possível adicionar serviços específicos diretamente em um Componente, sem afetar o plano padrão do Sistema.

Isso garante padronização, ao mesmo tempo que oferece flexibilidade para tratar exceções em equipamentos individuais.

O Ciclo de Trabalho Operacional

O uso do sistema foi projetado para se integrar ao fluxo de trabalho existente de planejamento e fiscalização, dividido em quatro fases principais.

Fase 1: Configuração

  • Carregar o arquivo de dados (.JSON) através da função "Abrir...".
  • Verificar se os Sistemas e Componentes estão corretos.
  • Realizar ajustes ou novos cadastros conforme a necessidade.

Fase 2: Planejamento

  • Na tela de "Relatórios", usar a função "Exportar Plano de Manutenção".
  • Definir o período e as áreas desejadas do plano de trabalho.
  • O arquivo gerado (.XLSX) é entregue para a equipe de execução.

Fase 3: Atualização

  • Após a execução, a contratada devolve o mesmo arquivo preenchido com os status.
  • O fiscal utiliza a função "Importar Relatório Técnico" para carregar o arquivo.
  • O sistema exibe uma tela de validação para o fiscal aprovar as atualizações, que são salvas em massa.

Fase 4: Análise

  • Acessar o "Dashboard" para uma visão do status geral dos serviços.
  • Utilizar a tela de "Indicadores" para análises sobre conformidade e backlog.
  • Consultar o histórico detalhado na tela de "Manutenções" usando os filtros avançados.

Capacidades Detalhadas do Sistema

Dashboard Central

Exibe o status atual dos serviços (Vencidos, Pendentes, A Vencer) e um calendário interativo para uma visão geral e rápida da programação.

Gestão de Status de OS

Permite o registro do ciclo de vida de cada serviço: conclusão, pendência com motivo, e reversão com justificativa para fins de auditoria.

Painel de Indicadores

Apresenta KPIs de desempenho (Taxa de Conformidade, Backlog) e gráficos sobre a distribuição dos serviços por status, criticidade e área.

Módulo de Relatórios

Centraliza a importação e exportação de dados em .XLSX para gerar planos de trabalho, documentação do PMOC e backups completos do histórico.

Consultas e Filtros

Oferece um sistema de filtragem multicritério para consultar o histórico de serviços por status, período, equipamento ou qualquer combinação de dados.

Gestão de Dados Local

Opera localmente (standalone), com os dados gerenciados pelo usuário através de arquivos .JSON por meio das funções de Abrir e Salvar.

Limitações Atuais e Boas Práticas

  • Operação Local (Standalone)

    Boa Prática: O sistema salva os dados no navegador, mas é fundamental usar a função "Salvar Como..." regularmente para criar backups do arquivo de dados (.JSON) em um local seguro (ex: próprio computador, rede corporativa, nuvem).

  • Sistema Monousuário

    Boa Prática: A ferramenta deve ser operada por um fiscal ou equipe de forma centralizada. Se diferentes pessoas precisarem atualizar os dados, devem fazê-lo a partir do arquivo .JSON mais recente para evitar conflitos de versão.

Primeiros Passos no Sistema

1. Carregue um PMOC

No topo da tela, clique em "Abrir..." e selecione o arquivo de dados (.JSON) com o plano de manutenção que deseja gerenciar.

2. Explore o Sistema

Navegue pelas seções do menu lateral. Comece pelos "Cadastros" para entender a estrutura e depois vá para "Manutenções" para ver os serviços agendados.

3. Simule o Fluxo

Vá para a seção "Relatórios" e tente exportar um plano de trabalho para o próximo mês. Em seguida, atualize o status de um serviço manualmente na tela de "Manutenções" para ver o sistema em ação.
